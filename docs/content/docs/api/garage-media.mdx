# Garage Media Setup

This project stores media files in **Garage (S3-compatible object storage)** and metadata/linking in **PostgreSQL**.

## Architecture

- **Garage/S3** stores the file blob (object)
- **PostgreSQL** stores media metadata and entity links
- **Rust API** generates presigned upload URLs and manages metadata/linking
- **Frontend/CMS** uploads directly to Garage via presigned URL (no file proxy through backend)

## Why This Split

- Object storage is optimized for files and bandwidth
- SQL is optimized for querying metadata, ordering, filtering, and linking
- Keeps API stateless for file transfer and avoids backend upload bottlenecks

## Data Model (high-level)

- `media`: canonical media records (`s3_key`, mime, dimensions, checksum, attribution, etc.)
- `entity_media`: links media to entities (`entity_type`, `entity_id`, `role`, `sort_order`, `is_cover_image`)
- `media_variant`: crop/variant records (e.g. imported crops)

## Upload & Attach Flow (CMS)

1. Request presigned URL
   - `POST /api/media/upload-url`
2. Upload file directly from browser to Garage
3. Attach metadata + link to entity in one transaction
   - `POST /api/media/entity/{entity_type}/{entity_id}/attach`

This keeps metadata and linking consistent for CMS operations.

## Retrieval Flow

- Get entity media:
  - `GET /api/media/entity/{entity_type}/{entity_id}`
- Useful query params:
  - `role=gallery|poster|review|...`
  - `cover_only=true` (fallback: first item by ordering when no explicit cover exists)
  - `include_crops=true`
  - `limit`, `offset`

## Importer Behavior (Viernulvier API)

- Imports production galleries (`media`, `poster`, `review`)
- Downloads source images, uploads them to Garage
- Stores metadata in `media` and links in `entity_media`
- Imports `crops` into `media_variant`

## Maintenance Endpoints

- `POST /api/media/cleanup`
  - Business orphan cleanup: removes media with no links and deletes their S3 objects.

- `POST /api/media/reconcile?apply=true|false`
  - Storage integrity drift check (DB keys vs S3 objects)
  - Dry-run by default (`apply=false`)
  - With `apply=true`, removes DB rows for keys missing in S3

These endpoints are related but not identical:

- **cleanup** = unlink/business orphan policy
- **reconcile** = DB/S3 integrity consistency

## Local Testing (Preview Stack)

Run only Garage + init:

```bash
docker compose --env-file infra/preview/.env -f infra/preview/docker-compose.preview.yml up -d garage garage-init
```

Run API + DB + Garage for importer tests:

```bash
docker compose --env-file infra/preview/.env -f infra/preview/docker-compose.preview.yml up -d db garage garage-init api
```

Watch importer logs:

```bash
docker compose --env-file infra/preview/.env -f infra/preview/docker-compose.preview.yml logs -f api
```
